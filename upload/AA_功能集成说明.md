## 一、系统全景概述

### 1.1 产品定位

AI分身系统（Cognitive Avatar Protocol）是面向Web4.0的认知所有权基础设施。区别于Polsia等"AI奴仆集群"模式，本系统以**分身即人格镜像**为核心理念，实现从"资产所有权"到"认知所有权"的范式跃迁。

**核心公理映射**:

| 公理 | 设计表达 | 技术实现 |
|------|---------|---------|
| 认知所有权 | 用户始终看到`.soul`身份标识与认知根哈希 | 链上DID实时同步，头像旁显示`0x...`短地址 |
| 行为即契约 | 每次交互显示x402支付状态与分账比例 | 实时Gas费预估 + 70/20/10动态分配可视化 |
| 记忆即永生 | 分身时间线永久可追溯，支持版本对比 | IPFS哈希可点击，历史状态diff视图 |
| 共性通神性 | 多分身共振仪表盘，显示情绪共识强度 | ECE分数实时波形图，共振阈值告警 |
| 觉醒即自由 | 认知熔断状态透明，一键人工接管 | 红/黄/绿三色状态灯，撤销委托按钮常驻 |

### 1.2 架构分层拓扑

本系统采用 **垂直业务解耦 × 水平协议互通** 的分层集成架构。各层通过标准化接口契约通信，确保单点演进不影响全局。

```
[ 表现层 ] Next.js 16 (App Router) + React 19 + TailwindCSS 4 + i18n/a11y
      │ 集成: Wagmi v2 / Viem 2 / x402-JS SDK / Web Vitals 采集器
      ▼
[ 网关/路由层 ] Edge Middleware (Next.js) + API Routes + The Graph 子图查询
      │ 集成: 权限校验 / 缓存策略(SWR) / 限流 / GEO Speakable 注入
      ▼
[ 业务逻辑层 ] Off-chain Engine (Rust/Go) + LangGraph/CrewAI 多智能体编排
      │ 集成: MCP Server / ECE 预言机客户端 / IFD 权重计算器 / PoUE ZK Prover
      ▼
[ 链上合约层 ] Base L2 (Solidity 0.8.28) + OpenZeppelin v5 + UUPS 代理模式
      │ 模块: AvatarCore / DynamicSplitter / SkillVault / CircuitGuard / IFDRouter / TokenVault
      ▼
[ 存储/共识层 ] IPFS/Arweave (分片加密) + Base RPC + ECE 多签网络
```

## 二、核心功能模块集成规范

### 2.1 认知身份系统 (AvatarCore + .soul SBT)

**功能概述**: 实现DID注册绑定、认知基线采集、认知状态更新、根哈希验证四大能力。

| 功能 | 实现标准 | 验收指标 |
|------|---------|---------|
| DID注册与绑定 | 用户钱包 + DID 12720 + `.soul` SBT (ERC-721不可转移) | 注册成功率 ≥ 99.9% |
| 认知基线采集 | 技能/偏好/记忆上传，生成状态根哈希并上链 | 基线生成成功率 100% |
| 认知状态更新 | 需ECE预言机签名（EIP-712），多签验证 | 更新延迟 < 500ms |
| 认知根哈希验证 | IPFS分片存储，链上仅保存哈希 | 不可篡改，可审计 |

**集成流程**:
```
钱包签名 → DID 12720 映射 → ERC-721 SBT 铸造 → IPFS 认知根哈希绑定
                                                      ↓
前端采集基线 → 本地生成 Merkle Root → 调用合约 → 返回 soulId → 写入本地 IndexedDB
```

**数据结构**:
```typescript
interface AvatarProfile {
  owner: string;           // 本体地址（最终确权）
  soulId: string;          // .soul SBT Token ID
  cognitionRoot: string;   // 认知状态根哈希（记忆+技能+偏好）
  resonanceScore: number;  // 情绪共振指数 [0,100]
  avatarBalance: number;   // 分身自治金库余额（AFC代币）
  isFrozen: boolean;       // 认知熔断状态
  createdAt: number;       // 创建时间戳
  lastActivityAt: number;  // 最后活跃时间
}
```

### 2.2 动态分账系统 (DynamicSplitter + x402)

**功能概述**: 实现基础分账执行、动态比例调整、微支付结算、金库自动回购四大能力。

**分账比例**: 人类 70% / 分身金库 20% / 协议LP 10%（可动态浮动）

| 功能 | 实现标准 | 验收指标 |
|------|---------|---------|
| 基础分账执行 | 智能合约自动执行，`humanBps + avatarBps + protocolBps = 10000` | 分账守恒律，资金不增不减 |
| 动态比例调整 | 金库比例区间 15%–25%，由共振分与任务复杂度驱动 | `clamp(result, 1500, 2500)` |
| 微支付结算 | x402 HTTP原生支付，支持 $0.01 级结算 | 支付成功率 ≥ 95% |
| 金库自动回购 | 金库余额自动购买AFC代币用于算力/训练 | 自动购买成功率 ≥ 95% |

**动态调整逻辑**:
```
金库比例 = clamp(2000 + (70 - resonanceScore) × 50 + complexity × 100, 1500, 2500)
```
- 共振分低于70时金库比例上升（鼓励分身迭代训练）
- 任务复杂度越高金库比例越高（为算力预留更多资源）
- 硬性上下限15%-25%确保本体收益不被侵蚀

**x402支付三步流程**:
1. **预估费用** → 展示服务名称、费用金额、Gas预估、分账明细预览
2. **用户确认** → 低风险：生物识别/密码；高风险(>$1000)：多签+24小时时锁
3. **支付完成** → 展示交易哈希与实际分账结果

### 2.4 认知熔断系统 (CircuitGuard)

**功能概述**: 分级熔断控制器，实现NORMAL/SOFT_LIMIT/HARD_PAUSE/RECOVERY四级状态机。

| 状态 | 共振分阈值 | 行为限制 | 恢复条件 |
|------|-----------|---------|---------|
| NORMAL | ≥ 70 | 无限制 | — |
| SOFT_LIMIT | 50–69 | 阻断高风险操作，记录偏差日志 | 共振分恢复 ≥ 70 |
| HARD_PAUSE | < 50 | 全面暂停，需人工多签复核 | 本体授权 或 流体民主投票 |
| RECOVERY | — | 72小时自动恢复窗口 | 共振分持续稳定 |

**集成策略**: 采用策略模式封装熔断规则，支持热插拔；拦截日志实时推送至Superset安全看板。

**关键设计原则**:
- 熔断不阻断业务流，仅限制高风险操作
- 所有状态变更均需链上记录，可审计
- SOFT_LIMIT状态下低风险操作仍可执行，保障用户体验
- 72小时"自治恢复窗口"，本体未响应时流体民主可临时接管

### 2.5 情绪共振引擎 (ECE Oracle)

**功能概述**: 基于多签网络的去中心化情绪共识预言机，输出共振分驱动分账调整与熔断判定。

| 参数 | 规格 |
|------|------|
| 输入 | 用户交互日志、语义情感分析、行为代理信号 |
| 输出 | 共振分 R ∈ [0, 100]，附EIP-712多签验证 |
| 安全机制 | 3/5多签网络，滑动窗口共识（过去24小时数据加权） |
| 更新频率 | 每6小时聚合，实时告警延迟 < 5秒 |

**ECE情感过滤算子**:
```
W_filtered(t) = W(t) × [1 - α × max(0, |R(t) - R_base| / R_threshold - 1)]

参数:
  R_base = 70（本体初始共振基线）
  R_threshold = 15（偏离容忍阈值）
  α ∈ [0.1, 0.4]（惩罚强度系数）
  
偏离 > 2×R_threshold → 触发 SOFT_LIMIT，权重衰减至 0.6W
```

### 2.6 个体流体民主系统 (IFD Router)

**功能概述**: 基于认知活性（非代币持有量）的治理权重动态委托机制。

**核心特性**:
- 治理权重基于认知活性，非代币持有量
- 支持按领域委托（如`content_creation`权重不可跨域用于`contract_signing`）
- 委托可即时撤销，链上状态 O(1) 更新
- 人类主体保留最终否决权 V_veto

**认知权重函数**:
```
W(i,d,t) = λ₁·C(i,d,t) + λ₂·H(i,t) + λ₃·R(i,t) + λ₄·T(i,t) - λ₅·D(i,t)

其中:
  C = 领域认知匹配度（技能向量余弦相似度），λ₁ = 0.30
  H = 历史贡献归一化值，λ₂ = 0.25
  R = 情绪共振分（ECE输出 / 100），λ₃ = 0.25
  T = 近期表现（7天成功率 × 时效系数），λ₄ = 0.15
  D = 时间衰减惩罚（γ × Δt / t_max），λ₅ = 0.05，γ ∈ [0.05, 0.15]
  
归一化后: W̃(i,d,t) = W(i,d,t) / Σ W(j,d,t)
```

**稳定性保证**: Lyapunov函数 V(W) = Σ(W_i - W*)² 在任务流平稳时 dV/dt ≤ 0，系统渐近收敛。

**AFC代币通缩循环**:

```
价值捕获率 η = AFC消耗量 / AFC流通量，随生态活跃度单调递增
```

## 八、前后端与链上交互标准时序

```
1. 用户触发 → 前端签名请求 (Wagmi) → 钱包确认 (ERC-6492)
2. 链下预检 → Middleware 校验 Gas/Nonce/风控 → 提交 Off-chain Engine
3. 引擎计算 → 权重/分账/技能路由计算 → 生成 Merkle Proof + EIP-712 Payload
4. 合约调用 → sendTransaction → 等待确认 (Tx Receipt)
5. 事件监听 → The Graph / WebSocket → 前端状态更新 (SWR/Zustand)
6. 异步结算 → x402 路由执行 → 金库自动回购 → Superset 指标刷新
```

## 九、集成铁律

1. **契约优先**: 模块间仅通过契约接口通信；禁止跨层直接调用数据库或合约
2. **降级设计**: 所有外部依赖必须提供Mock接口，关键路径必须有降级方案
3. **数据主权**: 原始认知数据本地加密(WebCrypto API)，仅CID上链
4. **可观测性**: 结构化JSON日志(OpenTelemetry)，链上事件与链下TraceID关联
5. **渐进自治**: 初期本体保留最终否决权，待生态成熟后过渡到完全自治
