## 一、核心功能模块清单

### 1.1 认知身份系统

| 功能 | 实现标准 | 验收指标 |
|------|---------|---------|
| DID注册与绑定 | 用户钱包 + DID 12720 + `.soul` SBT (ERC-721不可转移) | 注册成功率 ≥ 99% |
| 认知基线采集 | 技能/偏好/记忆上传，生成状态根哈希并上链 | 基线生成成功率 100% |
| 认知状态更新 | 需ECE预言机签名（EIP-712），多签验证 | 更新延迟 < 500ms |
| 认知根哈希验证 | IPFS分片存储，链上仅保存哈希 | 不可篡改，可审计 |

### 1.2 动态分账系统

分账比例：**人类 70% / 分身金库 20% / 协议LP 10%**（可动态浮动）

| 功能 | 实现标准 | 验收指标 |
|------|---------|---------|
| 基础分账执行 | 智能合约自动执行，`humanBps + avatarBps + protocolBps = 10000` | 分账守恒律，资金不增不减 |
| 动态比例调整 | 金库比例区间 15%–25%，由共振分与任务复杂度驱动 | `clamp(result, 1500, 2500)` |
| 微支付结算 | x402 HTTP原生支付，支持 $0.01 级结算 | 支付成功率 ≥ 95% |
| 金库自动回购 | 金库余额自动购买AFC代币用于算力/训练 | 自动购买成功率 ≥ 95% |

### 1.3 技能解锁系统（SkillVault）

| 收益门槛（USD等价） | 解锁内容 |
|-----------------|---------|
| $100 | 基础技能包（文案/客服/数据分析） |
| $500 | 高级RAG与检索增强 |
| $1,500 | 多模态生成（图像/视频） |
| $5,000 | 跨分身协作与联合推理 |

- 解锁触发：`require(cumulativeRevenue[soulId] >= revenueThresholds[tier])`
- 解锁后自动注册MCP/LangGraph新能力节点

### 1.4 认知熔断系统（CircuitGuard）

| 状态 | 共振分阈值 | 行为限制 | 恢复条件 |
|------|-----------|---------|---------|
| NORMAL | ≥ 70 | 无限制 | — |
| SOFT_LIMIT | 50–69 | 阻断高风险操作，记录偏差日志 | 共振分恢复 ≥ 70 |
| HARD_PAUSE | < 50 | 全面暂停，需人工多签复核 | 本体授权 或 流体民主投票 |
| RECOVERY | — | 72小时自动恢复窗口 | 共振分持续稳定 |

- 熔断不阻断业务流，仅限制高风险操作
- 所有状态变更均需链上记录，可审计

### 1.5 情绪共振引擎（ECE Oracle）

- **输入**：用户交互日志、语义情感分析、行为代理信号
- **输出**：共振分 R ∈ [0, 100]，附EIP-712多签验证
- **安全机制**：3/5多签网络，滑动窗口共识（过去24小时数据加权）
- **更新频率**：每6小时聚合，实时告警延迟 < 5秒

### 1.6 个体流体民主系统（IFD Router）

- 治理权重基于认知活性，非代币持有量
- 支持按领域委托（如`content_creation`权重不可跨域用于`contract_signing`）
- 委托可即时撤销，链上状态 O(1) 更新
- 人类主体保留最终否决权 V_veto

## 二、用户界面功能要求（D1设计规范）

### 2.1 核心界面组件

| 组件 | 功能 | 关键展示信息 |
|------|------|------------|
| 认知身份卡片 | 展示分身基本信息 | `.soul`标识、钱包地址、共振分、认知根哈希 |
| 动态分账仪表盘 | 可视化70/20/10分配 | 本月收益、各方金额、共振影响说明 |
| 情绪共振波形图 | 实时ECE数据可视化 | 24小时共振曲线、阈值线、当前分值与趋势 |
| 认知熔断面板 | 状态可视化与操作 | 三色状态灯（绿/黄/红）、操作日志、恢复按钮 |
| 流体民主委托界面 | 权重分配操作 | 领域选择、滑动调权、当前委托状态 |
| 技能库 | 展示已解锁/可解锁技能 | 解锁进度条、使用次数、满意度、费用 |
| 认知时间线 | 永久记忆可追溯 | 按时间排列的操作记录，含链上哈希 |

### 2.2 x402支付流程UI

三步流程：**预估费用 → 用户确认 → 支付完成**

- 预估页需展示：服务名称、费用金额、Gas预估、分账明细预览
- 低风险操作：生物识别/密码确认；高风险操作（>$1000）：多签+24小时时锁
- 支付完成后展示交易哈希与实际分账结果

### 2.3 国际化与可访问性标准

- 支持语言：zh-CN、zh-TW、en-US、ja-JP、ko-KR、es-ES、fr-FR、de-DE
- 无障碍：符合 WCAG 2.1 AA 标准
- 响应式：移动端（<768px）单列布局，桌面端（>1024px）三列布局
- 性能标准：FCP < 1.5s，LCP < 2.5s，INP ≤ 200ms，CLS < 0.1

## 三、GEO（生成式引擎优化）功能集成

### 3.1 Speakable Schema标注（P0优先级）

- 分身输出内容自动提取3–4个高信息密度段落
- 为每个片段添加语义标签（`skill_definition` / `use_case` / `evidence` / `tutorial`）
- 链上记录被引用次数，用作权威信号

### 3.2 引用即结算（行为即契约的延伸）

- 当分身内容被AI引擎或其他分身引用时，自动触发x402微支付（默认 $0.01）
- `citationCount`上链记录，达到5次自动进入领域推荐索引

### 3.3 性能监控（P1优先级）

- 前端采集INP/LCP/CLS，经差分隐私处理后提交链下预言机
- ECE引擎将`INP ≤ 200ms`作为共振分 +5 的校准因子
- 原始交互数据不离端，链上仅存聚合指标
